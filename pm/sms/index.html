📋 Overview

Executive Summary

SMS (Social Media Scheduler) is a multi-tenant SaaS platform solving the operational bottleneck of manual social media publishing for creators and agencies. This document covers the full product lifecycle: from problem discovery and user research through system architecture, API design, and delivery.

Purpose

Enable creators and agencies to manage every stage of their content lifecycle — from idea to publication to analytics — without manual intervention. SMS removes the need to publish content manually, remember posting schedules, or switch between multiple tools.

🎨 UX / Design

Wireframes

Lo-fidelity wireframes establish layout, hierarchy, and interaction patterns before any visual design is applied. These inform component specification and developer handoff.

Design Progression: Lo-Fi Wireframe → Hi-Fi Prototype → Final UI. The wireframes below represent the Lo-Fi stage — focused on structure and flow, not visual polish.

Dashboard — Lo-Fi Wireframe

Key decisions: 4-card stat row at top for immediate health check → chart for trend → list for upcoming actions → quick CTAs for immediate next step.

Scheduler — Lo-Fi Wireframe

Two-panel layout: creation form (65%) + live Instagram preview (35%). The preview pane is the most-requested feature from user testing — eliminates the need to post and delete to check appearance.

⚖️ Design Decision — Why a Two-Panel Scheduler?

Single-panel schedulers (like Buffer) require users to mentally simulate how their post will look. In user tests, 7/10 users asked "what will it actually look like on Social?" before scheduling. The live preview panel directly addresses this anxiety and reduces abandon rate during post creation.

Frequently Asked Questions

Q: What happens if a user's Social OAuth token expires while posts are scheduled?
A: The background worker will detect the invalid token, mark the job as Failed, and send an immediate email/in-app notification to the user to reconnect their account.

Q: How are daylight saving time (DST) changes handled?
A: Since we store publish times in UTC and use the server's IANA timezone library to convert from the user's selected timezone, DST shifts are handled automatically without user intervention.

Q: Can users publish Carousels?
A: Yes. The UI allows uploading up to 10 images/videos per post. The publishing worker uses the Platform APIs's carousel endpoints to group the media.
