Inventory Management System — Design and Data Flow

Design patterns and ERD for inventory systems: stock keeping, reservations, reconciliation, and integrations with order systems.

ERD (example)

Key entities: products, warehouses, stock_levels, stock_reservations, movements.

-- Simplified ERD tables
CREATE TABLE products (id UUID PRIMARY KEY, sku TEXT UNIQUE, name TEXT);
CREATE TABLE warehouses (id UUID PRIMARY KEY, name TEXT);
CREATE TABLE stock_levels (product_id UUID REFERENCES products(id), warehouse_id UUID REFERENCES warehouses(id), available INT, reserved INT, PRIMARY KEY(product_id, warehouse_id));
CREATE TABLE stock_reservations (id UUID PRIMARY KEY, product_id UUID, warehouse_id UUID, quantity INT, status TEXT, created_at TIMESTAMP);

Data flow & reconciliation

  • Event-driven updates for stock movements (message queue → consumers → update aggregate views)
  • Periodic reconciliation jobs to fix drift and expired reservations
  • Optimistic concurrency on reservations with compensation flows on conflicts
