Building a Real-Time Chat System (WhatsApp-like)

Patterns for real-time messaging: connection management, delivery guarantees, replication, presence, and storage trade-offs.

Core components

  • Connection layer (WebSocket/QUIC) for persistent connections
  • Message broker for fanout and durable delivery (Kafka, NATS Streaming)
  • Delivery semantics: at-most-once, at-least-once, exactly-once (trade-offs)
  • Storage: hot store for recent messages, cold store for archives

Scaling

  • Partition conversation space by conversation id
  • Use presence service for routing to active connections
  • Design for eventual consistency across multi-device delivery
