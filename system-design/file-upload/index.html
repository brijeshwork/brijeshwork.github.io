Designing a File Upload and Media Management System

Patterns for secure, reliable file uploads, object storage, CDN integration, processing pipelines, and cost-effective media delivery.

Requirements

  • Support large files and resumable uploads
  • Process images/videos (thumbnails, transcoding) asynchronously
  • Deliver via CDN with cache invalidation support
  • Apply access controls and virus scanning

Architecture notes

  • Pre-signed upload URLs → direct-to-storage uploads (S3/GCS)
  • Use message queue to trigger processing workers
  • Store metadata in DB and objects in an object store
  • CDN in front for fast global delivery
