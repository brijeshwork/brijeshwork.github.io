Designing a Search Engine System (Indexing + Ranking)

Architectural overview for building search: crawler, indexer, inverted index, query processing, ranking and relevance tuning.

Components

  • Crawler to discover content and follow sitemaps
  • Indexer to parse and create inverted index with term frequencies
  • Query engine for parsing queries and merging posting lists
  • Ranking layer with signals (TF-IDF, BM25, learning-to-rank)

Scaling & sharding

Shard index by document or term ranges; use replication for high availability and low-latency query serving.
